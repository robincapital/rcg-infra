## Why this exists
Before consolidation, the screener and the per-ticker report had two completely
different price-target engines. The screener's was missing 5 guardrails the
report had, and both were missing 2 critical gates. When a ticker passed the
screener, the report read `target_price` from the screener CSV — so the report's
better engine never ran on screener-included tickers. INGM published $474.53
target on a $30 stock with R²=0.000 on its dominant model, while analyst
consensus said $27.85.

## Three new gates (vs legacy)

### Gate A — R² floor on conviction
Old behavior: a model with R² = 0.00 still earned ~50% conviction via
stability and data-points terms.

New behavior:
| R² | Conviction |
|----|-----------|
| < 0.20 | **0** — model is dropped from the blend |
| 0.20 – 0.40 | linear ramp (0 → full) |
| ≥ 0.40 | full formula `0.5·R² + 0.3·stability + 0.2·n_score` |

Tunable via `PT_R2_FLOOR` and `PT_R2_FULL`.

### Gate B — Analyst-consensus envelope
Old: when model PT diverged from analyst by >40%, screener flagged with `M*`
but published the model PT unchanged.

New:
| Divergence (% of price) | Source label | Action |
|-------------------------|--------------|--------|
| ≤ 40% | `M✓` | Published unchanged |
| 40 – 75% | `M*` | Flagged but published |
| > 75% | `M⚠clip` | Clipped to `[analyst × 0.5, analyst × 1.5]` |

Triggers only when `n_analysts ≥ 3`. Tunable via `PT_APPLY_ENVELOPE`.

### Gate C — FCF runaway cap (added Day 2)
The FCF Yield model can produce PTs 5–10× current price when Theil-Sen
extrapolates a strongly-trending FCF series and divides by a small required
yield (~5%). ENVA's FCF model produced PT $1,705 for a $173 stock.

When at least one other valuation model survives, the FCF model PT is
capped at `min(4 × last_price, 2 × max(other surviving PTs))`. Sets
`applied_cap = "FCF_RUNAWAY_CAP"` and records the uncapped PT for audit.

### Carries over from the report's engine
- Quality-score haircut (0.40–1.00 multiplier on final blended PT)
- Growth-adjusted EV/Rev anchor (0.6× – 2.2× scalar)
- Mean-reversion cap on low-quality names (`quality < 0.60` → blend ≤ sector anchor)
- 2.5× sector ceiling on EV/Rev for healthy names
- Emerging Growth Model 4 (TAM-discounted projection at 25% PV rate)

### Carries over from the screener's engine
- Rate compression on sector multiples (Fed-rate conditional)

## Source labels (`pt_source`)

| Label | Meaning |
|-------|---------|
| `M` | Model — no analyst data available |
| `M✓` | Model agrees with analyst (within 40% of price) |
| `M*` | Model flagged but published (40–75% divergence) |
| `M⚠clip` | Model clipped to analyst envelope (>75% divergence) |
| `A` | All models dropped — falling back to analyst consensus |
| `N/A` | No PT available |

### Day 1 — Module shipped (2026-04-28)

### Day 2 — Regression harness validated gates (2026-04-28)
First run revealed two issues requiring immediate patches:

1. **n_analysts unreliable**. Finnhub's `/recommendation` endpoint covers fewer
   tickers than `/price-target`. Gate B was engaging on only ~30% of valid
   targets.

2. **FCF model PT runaway**. ENVA $1,705 PT, OPY $989, FRHC $1,599, DAVE $759.
   - Fix: Gate C added.

Final regression: 21 CHANGE / 12 OK / 7 FALLBACK_TO_ANALYST / 0 HALT.
Spot-check on PVH, FRHC, DAVE, REPX, BVS confirmed gates fire on the right
names for the right reasons.

### Day 3 — R² floor sweep deferred (2026-04-28)
- Fixed early-binding bug in `_model_conviction` (default args were freezing
  R²_HARD_FLOOR at module-load time).

  all floors and snapshots came out flat or slightly negative.
- **Decision:** Deferred. Floor locked at 0.20 based on Day 2 qualitative
